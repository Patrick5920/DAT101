1. Where should you use constants, and where should you use variables? Can they all be constants?

    2. Constants should be used for values that do not change during program execution (e.g., tax rates, PI, maximum limits). Variables are used for values that change (e.g., user input, counters, scores). They cannot all be constants, because programs usually need to update values as they run. Any value that must change cannot be a constant.

  2. How do parentheses affect the arithmetic outcome of an expression?

    3. Parentheses control the order of operations. Expressions inside parentheses are evaluated first, overriding normal precedence rules (PEMDAS).

  3. How do we limit floating-point numbers to a given number of decimals?

    4. To limit decimal places, use formatting functions (like toFixed(), f-strings, format()) for display to get strings (e.g., "3.14") or mathematical rounding (multiply, round, divide) for numerical results (e.g., 3.14).

  4. How can we compare two floating-point values to see if they are equal (is 12.199999 the same as 12.2)?

    5. Floating-point numbers can have precision errors, so direct comparison (==) is unreliable. Instead, compare the difference using a small tolerance (epsilon): If |a - b| < 0.0001, treat them as equal. So 12.199999 and 12.2 are effectively equal within a tolerance.

  5. What do comparison operators return, and how can we check if they are FALSE or TRUE?

    6. Comparison operators (<, >, ==, !=, <=, >=) return Boolean values: true or false We can check them using: if statements Boolean expressions Example: 5 > 3 returns true
