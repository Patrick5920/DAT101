  2. 2. In part 2, can we create an “infinite” loop within the condition of the while loop?

    3. Yes, it is possible to create an infinite loop if the condition of the while loop never becomes false. This can happen if the guessed number is not updated inside the loop or if the random number can never match the target value. To prevent this, the guessed number must change on each iteration and the loop condition must eventually be satisfied.

  3. 3. How quickly or slowly can a computer compute this, and how does the program behave while guessing?

    4. A computer can perform the guessing very quickly, even if many guesses are required, because each operation takes very little time. While the program is guessing, it executes the loop repeatedly and does not perform other tasks at the same time in a single-threaded environment. The program continues running the loop until the correct number is found.

  4. 4. Explain the relationship between rows and columns in nested for loops.

    5. In nested for loops, the outer loop usually represents the rows and the inner loop represents the columns. For each iteration of the outer loop, the inner loop runs completely. The order of rows and columns can be changed, but doing so affects how the data is processed and displayed.
