1. 1. What is the difference between invoking and creating a function?

    2. Creating (defining) a function means writing the function code. Invoking (calling) a function means executing it. A function can be created once and invoked many times.

  2. 2. Explain the syntax regarding semicolons and code blocks

    3. Semicolons (;) are used to end individual statements. Code blocks are enclosed in {} and group multiple statements together. Semicolons are not used after code blocks like functions or loops.

  3. 3. What happens when we try to store values from a function that doesnâ€™t return any?

    4. If a function has no return statement, it returns undefined by default. Storing its result will store undefined. This can cause errors if the value is later used in calculations.

  4. 4. How do we separate parameters from each other when creating a function?

    5. Parameters are separated by commas. Example: function example(a, b, c)

  5. 5. How are parameter names related during invocation and creation of a function?

    6. Parameter names in the function definition are placeholders. When the function is invoked, the arguments are matched by position, not by name. The names used during invocation do not need to match the parameter names.

  6. 6. Why is a parameter undefined when a function call is missing a parameter?

    7. If an argument is not provided, the parameter has no assigned value. In JavaScript, missing parameters default to undefined. This can lead to unexpected behavior unless default values are used.

  7. 7. What happens when we use recursive functions?

    8. A recursive function calls itself. Each call adds a new frame to the call stack. Recursion must include: A base case (to stop recursion) A recursive case (to continue) Without a base case, recursion leads to stack overflow errors.
